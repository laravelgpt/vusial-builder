Introduction

Laravel Visual Builder is a powerful package for building and managing visual components, AI agents, security features, and cloud storage integrations in your Laravel applications.

Quick Start View on GitHub

Core Features

Visual Component Builder

Visually create, customize, and manage your Laravel components with an intuitive drag-and-drop interface. Streamline your development workflow and build beautiful UIs faster than ever.

AI Agent Integration

Seamlessly integrate powerful AI agents into your application for intelligent automation, content generation, and advanced data analysis.

Robust Security & Middleware

Protect your application with built-in security features, including middleware control, input sanitization, and authorization through Laravel's gates and policies.

Dynamic Real-Time UI

Build reactive and dynamic interfaces with deep integration for Laravel Livewire and real-time updates powered by Laravel Reverb.

Flexible Storage Options

Easily configure your components to use different storage disks, from local filesystems to cloud providers like Amazon S3.

Advanced Developer Tools

Fine-tune your application with powerful developer-focused tools like feature flags for gradual rollouts and a centralized visualization configuration.

Advanced Media Library

Manage your media files with a powerful, integrated library that supports various file types, conversions, and storage options.

Role-Based Access Control

Integrates with Laravel's native permission system, allowing you to control who can access the builder and manage specific components.

Automated Backups

Configure and manage automated backups for your application's database and files, storing them securely on any supported storage disk.

Extensible Plugin Architecture

Develop your own plugins and extensions to add custom functionality, components, and integrations to the Visual Builder.

Advanced Theming

Go beyond basic styling with a deep theming system that allows for custom color palettes, typography, and component-level design tokens.

Component Versioning

Keep track of component changes with a built-in version history. Roll back to previous versions or compare changes directly within the builder.

Powerful CLI Tools

Scaffold new components, manage configurations, and run build processes directly from the command line with custom Artisan commands.

Installation

Install the package via Composer:

composer require laravel-builder/visual-builder

You can publish the configuration file with this command:

php artisan vendor:publish --provider="LaravelBuilder\VisualBuilder\VisualBuilderServiceProvider" --tag="config"

This is the main configuration file for the Visual Builder, where you can set up middleware, component paths, and more. For advanced customization, you can also publish other resources:

# Publish the views
php artisan vendor:publish --provider="LaravelBuilder\VisualBuilder\VisualBuilderServiceProvider" --tag="views"

# Publish the JavaScript assets
php artisan vendor:publish --provider="LaravelBuilder\VisualBuilder\VisualBuilderServiceProvider" --tag="assets"

# Publish the database seeders
php artisan vendor:publish --provider="LaravelBuilder\VisualBuilder\VisualBuilderServiceProvider" --tag="seeders"

After installation, make sure to run php artisan config:cache to cache the configuration files.

The Builder Interface

The heart of Laravel Visual Builder is its intuitive, user-friendly interface, designed to make page and component creation as seamless as possible. The interface is primarily composed of three key areas:

The Canvas

This is your main work area where you can see a live, interactive preview of the page or component you are building. You can directly manipulate elements on the canvas, drag and drop new components, and see changes in real-time.

Component Panel

Located on the side, this panel contains a library of all available components—from basic HTML elements to complex, data-driven Laravel components. It's organized by category for easy navigation. Any custom component you create is automatically registered here.

Property Inspector

When you select a component on the canvas, the inspector panel shows all its configurable properties (props). You can change text, colors, spacing, data bindings, and other settings here, with changes reflected instantly on the canvas.

Data Binding

The property inspector allows you to bind component properties to various data sources. You can link a property to a specific model attribute, a configuration value, or the result of a query. This makes creating dynamic, data-driven components incredibly efficient.

Custom Component Integration

To add your own component to the builder, you simply create a standard Laravel component. The Visual Builder automatically discovers any components in your `app/View/Components` directory and makes them available in the component panel. No extra configuration is needed.

Builder Services

The Visual Builder

The core of the package is the Visual Builder itself. It's a powerful and intuitive interface for creating, managing, and composing your UI. By combining pre-built components with your own custom creations, you can design complex interfaces with ease. It's designed to accelerate development by giving you a real-time preview of your application's frontend.

The AI Agent Builder

Leverage the AI Agent Builder to integrate intelligent features directly into your application. You can create and configure AI agents that perform tasks like content generation, data analysis, or interacting with users via a chat interface. This service connects to major AI providers and streamlines the process of adding machine learning capabilities to your project.

The Security Builder

The Security Builder provides a suite of tools and configurations to harden your application. It allows you to enforce security best practices, such as configuring middleware, setting up content security policies, and integrating malware scanners. This builder helps ensure that your application is not only functional but also secure from common threats.

Base Configuration

Configure the core settings of Laravel Visual Builder through the published configuration file:

// config/visual-builder.php

return [
    'prefix' => 'builder',
    'middleware' => ['web', 'auth'],
    'storage' => [
        'disk' => 'local',
        'path' => 'builder',
    ],
    'components' => [
        'namespace' => 'App\Components',
        'path' => app_path('Components'),
    ],
];

Visualization Settings

Configure the visual aspects of your components via the `config/visual-builder.php` file. You can define theme colors, fonts, and other styling options.

// config/visual-builder.php
'visualization' => [
    'theme' => 'default', // or 'dark'
    'colors' => [
        'primary' => '#3490dc',
        'secondary' => '#6574cd',
        'danger' => '#e3342f',
    ],
    'fonts' => [
        'body' => '"Nunito", sans-serif',
        'heading' => '"Montserrat", sans-serif',
    ],
    'spacing' => [
        'default' => '1rem',
        'large' => '2rem',
    ]
],

These settings can then be programmatically accessed within your components or services to apply consistent styling across your application, ensuring a cohesive user experience.

Livewire Integration

Integrate Livewire components with Laravel Visual Builder for dynamic, real-time user interfaces without leaving the comfort of PHP.

class DynamicForm extends Component
{
    use WithVisualBuilder;

    public $formData = [];

    public function render()
    {
        return view('livewire.dynamic-form');
    }

    public function submit()
    {
        $this->validate([
            'formData.email' => 'required|email',
            'formData.password' => 'required|min:8',
        ]);
        // Handle form submission
    }
}

Middleware Configuration

Protect your visual builder routes and components by applying middleware. You can define global middleware in your `config/visual-builder.php` file.

// In config/visual-builder.php
'middleware' => [
    'web',
    'auth',
    \App\Http\Middleware\CheckSubscription::class,
],

This ensures that only authenticated users with an active subscription can access the builder, providing a robust layer of security.

Feature Flags

Implement feature flags for gradual rollouts and A/B testing of your components. Laravel Visual Builder can be integrated with popular feature flag services or a simple config-based implementation.

use LaravelBuilder\VisualBuilder\Facades\Features;

if (Features::enabled('new-component-builder')) {
    // Show the new, enhanced component builder
    return view('components.new-builder');
} else {
    // Fall back to the legacy component builder
    return view('components.legacy-builder');
}

Real-time Updates (Reverb)

Integrate real-time functionality using Laravel Reverb for live previews, collaborative editing, or instant notifications.

First, ensure Reverb is installed and configured. Then, broadcast events from your backend when a component is updated.

// app/Events/ComponentUpdated.php
namespace App\Events;

use Illuminate\Broadcasting\Channel;
use Illuminate\Broadcasting\InteractsWithSockets;
use Illuminate\Contracts\Broadcasting\ShouldBroadcast;
use Illuminate\Foundation\Events\Dispatchable;
use Illuminate\Queue\SerializesModels;

class ComponentUpdated implements ShouldBroadcast
{
    use Dispatchable, InteractsWithSockets, SerializesModels;

    public function __construct(public array $component) {}

    public function broadcastOn(): array
    {
        return [new Channel('builder')];
    }
}

Then, listen for these events on your frontend:

// resources/js/echo.js
import Echo from 'laravel-echo';
import Pusher from 'pusher-js';

window.Pusher = Pusher;

window.Echo = new Echo({
    broadcaster: 'reverb',
    key: import.meta.env.VITE_REVERB_APP_KEY,
    wsHost: import.meta.env.VITE_REVERB_HOST,
    wsPort: import.meta.env.VITE_REVERB_PORT,
    wssPort: import.meta.env.VITE_REVERB_PORT,
    forceTLS: (import.meta.env.VITE_REVERB_SCHEME ?? 'https') === 'https',
    enabledTransports: ['ws', 'wss'],
});

window.Echo.channel('builder')
    .listen('ComponentUpdated', (e) => {
        console.log('Component updated:', e.component);
        // Add logic to refresh the component on the page
    });

Security Considerations

Security is paramount. While Laravel Visual Builder adheres to Laravel's security best practices, it's crucial to be mindful of security when creating components.

Input Sanitization & Validation

  • Always validate data submitted through forms or component properties using Laravel's Validator.
  • Sanitize user-generated content to prevent XSS. Blade's `{{ }}` syntax escapes by default.
  • Be cautious with ` {!! !!} ` syntax; only use it with trusted HTML.

Authorization

  • Use Laravel's policies and gates to control who can access the builder and manage specific components.
  • Apply appropriate middleware to routes (see Middleware section).

Malware Scanning (Conceptual)

If your components handle file uploads, consider integrating a malware scanner. This is an advanced topic and might involve third-party services or libraries.
